# お掃除ナビ 今後の展開：サイト機能・構成 × 集客・SEO

サイト機能・構成の強化と、集客・SEOの強化をまとめたロードマップです。  
**搭載済み**と**今後必要な機能**に分けて整理しています。

---

## 1. 搭載済み機能（現状）

### 1-1. サイト構造・ナビゲーション

| 項目 | 内容 |
|------|------|
| **パンくずリスト** | 全ページで「ホーム > 場所で探す」などの導線を表示（`Breadcrumbs.astro`）。記事・カテゴリ・トップで利用。 |
| **フッターのサイトマップ** | ホーム／場所／汚れ／季節／共通知識／ブログ一覧へのリンクを明示。 |
| **カテゴリ構成** | 場所で探す（`/place/`）・汚れで探す（`/stain/`）・季節で探す（`/season/`）・**共通知識**（`/basic/`）・ブログ（`/blog/`）・**人気記事ランキング**（`/likes/`）。 |

### 1-2. 記事・コンテンツ

| 項目 | 内容 |
|------|------|
| **記事の構造化データ** | 各記事に JSON-LD（Article）。`datePublished`／`dateModified`（`updatedDate` がある場合）を出力。 |
| **OGP・SNS** | 全ページで `og:image`（デフォルト `ogp.png`）・`og:title`・`og:description`・Twitter カードを設定。 |
| **公開日・更新日** | 記事ヘッダーに「公開日：〇〇」を表示。フロントマターに `updatedDate` があれば「最終更新：〇〇」も表示。 |
| **あわせて読みたい** | 記事本文直後に、`relatedArticles` の先頭1件へのリンクブロックを表示。 |
| **関連記事** | 記事末尾（最終パート）で、同じ場所・汚れ・季節の記事を最大3本表示。 |
| **記事共有** | X（Twitter）・Facebook・LINE・URLコピー（`ArticleShare.astro`）。 |

### 1-3. SEO・メタ情報

| 項目 | 内容 |
|------|------|
| **タイトル・description のキーワード** | トップ・場所／汚れ／季節／ブログ各ページの `title`・`description` に「お掃除 方法」「掃除 コツ」「場所別・汚れ別」などを含めて強化済み。 |
| **canonical・メタ** | 全ページで canonical・meta description・OGP 基本を設定。 |
| **サイトマップ・robots** | `sitemap-index.xml`、`robots.txt`（クロール許可＋サイトマップ参照）。 |
| **記事の title／description** | フロントマターで管理。新規・加筆時はキーワードを意識した文言を推奨。 |

### 1-4. 分析・エンゲージメント

| 項目 | 内容 |
|------|------|
| **Google アナリティクス（GA4）** | 本番環境で `PUBLIC_GA_MEASUREMENT_ID` を参照し、`gtag.js` で計測。 |
| **「なるほど！」ボタン** | 記事ページ下部に設置。Firebase（Firestore）でクリック数を保存。1ブラウザ1回までカウント。 |
| **人気記事ランキング** | `/likes/` で、Firestore の「なるほど」数に基づくランキングを表示。タイトルは Firestore またはビルド時の slug→title マップで表示。 |

### 1-5. コンテンツ・カテゴリ

| 項目 | 内容 |
|------|------|
| **共通知識カテゴリ** | `/basic/`。洗剤・道具の使い分けなど「共通の基本知識」をまとめた記事を `category: basic` で一覧。 |
| **まとめブログ** | 例：平日10分ミニ掃除ルーティン（`category: blog`）。場所別・季節別など既存記事への内部リンクを多数設置。 |
| **内部リンク例** | キッチン油汚れ記事などで、見出しにキーワードを含め、本文に他記事へのリンクを追加済み（加筆時に他記事も同様に増やしていく想定）。 |

## 2. 今後必要な機能・構成

### 2-1. 優先して取り組みたい

| 項目 | 内容 | 効果 |
|------|------|------|
| **サイト内検索** | キーワードで記事を検索（静的：全記事を JSON で出力しクライアントでフィルタ、または検索用静的ページ） | 欲しい情報にすぐ到達・滞在時間・回遊率アップ |
| **人気記事ブロック** | トップや記事下に「よく読まれている」ブロック（人気ランキング `/likes/` へのリンク、または Firestore の上位を取得して表示） | 内部リンク強化・初回訪問者の回遊 |
| **カテゴリページの説明文** | 場所／汚れ／季節／共通知識の各ページに、そのカテゴリの説明文（1〜2行）を追加 | SEO（キーワード・文脈）・信頼感 |
| **Search Console の活用** | インデックス状況・検索クエリ・クリック数を確認し、伸ばしたいキーワードを記事化・修正（運用） | 実データに基づく改善 |

### 2-2. 中期的に取り組みたい

| 項目 | 内容 | 効果 |
|------|------|------|
| **記事一覧の並び替え・表示数** | ブログ一覧で「新着」「人気」切替、または「もっと見る」で増量表示 | 使いやすさ・回遊 |
| **キーワード一覧の整備** | 狙うキーワードを一覧化し、どの記事でカバーするかマッピング | 被り・抜けの防止・新記事のテーマ決め |
| **ランディングページの整理** | 場所／汚れ／季節のトップを「〇〇の掃除まとめ」として説明＋記事リンクのまとめに | まとめ検索での露出・オーソリティ |
| **外部被リンク・露出** | Pinterest・まとめサイト・SNSなどで記事リンクを共有（無理のない範囲で） | 被リンク・直接流入 |
| **Core Web Vitals** | LCP・CLS などを Lighthouse で確認し、画像の遅延読み込みやレイアウト固定で改善 | 検索ランキング・体感速度 |

### 2-3. 記事まわりで継続したいこと

| 項目 | 内容 |
|------|------|
| **見出しにキーワード** | 加筆・新規時に「換気扇 掃除」「水垢 落とし方」などを H2/H3 に含める。既存記事は加筆時に順次調整。 |
| **本文中の内部リンク** | 関連記事へのリンクを1〜2本、アンカーにテーマが分かる語を含めて追加。 |
| **テンプレートの目安** | title は「〇〇の掃除｜△△を落とす方法」のようにキーワード＋ベネフィット。description は150字前後で主キーワードと「方法」「コツ」「手順」を含める。 |

## 3. 詳細メモ（SEO・運用）

### タイトル・description のキーワード（実施済み・追加時の参考）

- トップ：`お掃除の方法・掃除のコツ｜場所別・汚れ別のやり方`
- 場所で探す：`キッチン・浴室・リビングなど場所別の掃除方法`
- 汚れで探す：`油汚れ・水垢・カビの落とし方｜汚れ別の掃除方法`
- 季節で探す：`春・夏・年末など季節別のお掃除｜大掃除のコツ`
- ブログ：`お掃除の記事一覧｜掃除方法・掃除のコツ`
- 記事：主キーワード＋ベネフィット。description は150字前後で自然にキーワードを含める。詰め込みは避ける。

### Search Console で確認したいこと

- **パフォーマンス**：検索クエリ、表示回数、クリック数、平均掲載順位
- **カバレッジ**：インデックス状況・エラー有無
- **URL検査**：重要ページのインデックス登録リクエスト
- **活かし方**：表示多・クリック少 → タイトル・description 見直し。10位前後 → 見出し・本文を少し強化。記事がないキーワード → 新記事のテーマ候補に。週1回〜月1回の確認を推奨。

### 技術メモ

- **検索**：Astro は静的なので、(1) 全記事データを JSON で出力しクライアントでフィルタ、(2) 検索用の静的ページを複数生成、のいずれかが現実的。
- **人気記事**：Firestore の `articleLikes` を `orderBy('count','desc')` で取得する実装済み。トップ／記事下の「人気ブロック」はこのデータを再利用可能。

## 4. 次の一歩

- **すぐ着手しやすい**：カテゴリページへの説明文1〜2行、トップまたは記事下への「人気ランキングへ」リンクまたは人気ブロック。
- **少し時間をかける**：サイト内検索（クライアント検索 or 静的検索ページ）。
- **継続**：新規・加筆記事の見出し・本文にキーワードと内部リンクを意識、Search Console の確認とキーワード一覧の更新。
