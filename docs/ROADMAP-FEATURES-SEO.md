# お掃除ナビ 今後の展開：サイト機能・構成 × 集客・SEO

サイト機能・構成の強化と、集客・SEOの強化をまとめたロードマップです。  
**すぐできるもの**と**中期的に取り組むもの**に分けてあります。

---

## 1. サイト機能・構成の強化

### 1-1. すぐ実装しやすい（優先度高）

| 項目 | 内容 | 効果 |
|------|------|------|
| **パンくずリスト** | 全ページで「ホーム > 場所で探す > キッチン」のような導線を表示 | 離脱減・SEO（構造の理解）・内部リンク |
| **記事ページの構造化データ** | 各記事に JSON-LD（Article）を追加 | 検索結果でリッチスニペット（日付・説明）が出やすくなる |
| **OGP画像** | `og:image` を設定（トップ用＋記事用のデフォルト） | SNSシェア時のクリック率向上 |
| **フッターのサイトマップリンク** | 場所／汚れ／季節／ブログ一覧へのリンクを明示 | クロール・ユーザーの探しやすさ |

### 1-2. 中期的に追加したい

| 項目 | 内容 | 効果 |
|------|------|------|
| **サイト内検索** | キーワードで記事を検索（静的：全記事をJSでフィルタ or 検索ページを静的生成） | 欲しい情報にすぐ到達でき、滞在時間・回帰率アップ |
| **人気記事ブロック** | トップや記事下に「よく読まれている」（まずは新着と同じでOK、後でPVベースに） | 内部リンク強化・初回訪問者の回遊 |
| **カテゴリページの充実** | 場所／汚れ／季節の各ページに、そのカテゴリの説明文（1〜2行）を追加 | SEO（キーワード・文脈）・信頼感 |
| **記事一覧の並び替え・表示数** | ブログ一覧で「新着」「人気」切替、または「もっと見る」で増量表示 | 使いやすさ・回遊 |

### 1-3. 技術メモ

- **検索**：Astro は静的なので、検索は (1) 全記事データを JSON で出力しクライアントでフィルタ、(2) 検索用の静的ページを複数生成、のどちらかが現実的。
- **パンくず**：`BaseLayout` に props で `breadcrumbs: { label, href }[]` を渡し、記事・カテゴリ・トップで共通コンポーネントを使う。

## 2. 集客・SEOの強化

### 2-1. すでにできていること（維持）

- サイトマップ（`sitemap-index.xml`）
- `robots.txt`（クロール許可＋サイトマップ参照）
- 各ページの canonical・meta description・OGP 基本
- 記事の `title` / `description` をフロントマターで管理

### 2-2. すぐ強化できる（優先度高）

| 項目 | 内容 | 効果 |
|------|------|------|
| **タイトル・description のキーワード** | トップ・カテゴリ・記事で「お掃除 方法」「〇〇 掃除」など検索されそうな語を自然に含める | 検索キーワードとの一致・CTR向上 |
| **見出し（H2/H3）にキーワード** | 記事内の見出しに「換気扇 掃除」「水垢 落とし方」などを含める（既存記事は加筆時に調整） | 検索エンジンがテーマを理解しやすくなる |
| **内部リンクの強化** | 関連記事を「同じ場所・汚れ・季節」で表示（現状あり）。加えて、記事本文中に「キッチンの水垢は別記事で」などリンクを増やす | クロール・権威の分散・回遊 |
| **更新日の明示** | 記事に `updated` または「最終更新」を表示（まずは `pubDate` でも可） | 鮮度シグナル・ユーザーへの説明 |
| **Search Console の活用** | インデックス状況・検索クエリ・クリック数を確認し、伸ばしたいキーワードを記事化・修正 | 実データに基づく改善 |

### 2-3. 中期的に取り組む

| 項目 | 内容 | 効果 |
|------|------|------|
| **キーワード一覧の整備** | 「お掃除 やり方」「換気扇 掃除 重曹」など、狙うキーワードを一覧化し、どの記事でカバーするかマッピング | 被り・抜けの防止・新記事のテーマ決め |
| **ランディングページの整理** | 場所／汚れ／季節のトップを「〇〇の掃除まとめ」として説明＋記事リンクのまとめページに | まとめ検索での露出・オーソリティ |
| **外部被リンク・露出** | Pinterest・まとめサイト・SNSなどで記事リンクを共有（無理のない範囲で） | 被リンク・直接流入 |
| **Core Web Vitals** | LCP・CLS などを Lighthouse で確認し、画像の遅延読み込みやレイアウト固定で改善 | 検索ランキング・体感速度 |

### 2-4. 記事テンプレートで揃えたいこと（新規・加筆時）

- **title**：できれば「〇〇の掃除｜△△を落とす方法」のようにキーワード＋ベネフィット。
- **description**：150字前後で、主キーワードと「方法」「コツ」「手順」などを含める。
- **本文**：最初の段落に主キーワードを1回、見出しに関連キーワードを含める。

## 3. 実装の優先順位（提案）

1. **すぐやる（コスト低・効果大）**  
   - パンくずリスト  
   - 記事の JSON-LD（Article）  
   - OGP画像（デフォルト1枚でOK）  
   - トップ・カテゴリの title / description のキーワード見直し  

2. **1〜2週間で**  
   - サイト内検索（シンプルなクライアント検索）  
   - フッターのサイトマップリンク  
   - カテゴリページに短い説明文追加  

3. **継続**  
   - 新規記事の見出し・本文にキーワードを意識  
   - Search Console の確認とクエリに合わせた修正  
   - キーワード一覧の更新
